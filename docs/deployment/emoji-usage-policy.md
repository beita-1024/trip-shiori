# 絵文字使用方針

このプロジェクトでは、ログ出力やメッセージ表示における絵文字の使用を制限し、統一性を保つための方針を定めています。  
**理由**:  
- 文字化けしにくい  
- 幅が安定している（ズレが起きにくい）  
- 合成絵文字ではないため、環境ごとのレンダリング差分が出にくい  

## 使用可能な絵文字

以下の3つの絵文字のみを使用可能とします：

- **❌** - エラー・失敗を示す
- **✅** - 成功・完了を示す  
- **⚠️** - 警告・注意を示す

## 使用原則

### 1. 過剰な使用を避ける
- 絵文字は重要な状態を示す場合のみ使用
- 装飾目的での使用は禁止
- 1つのメッセージに複数の絵文字を使用しない

### 2. 適切な使用場面
- **❌**: エラーメッセージ、失敗時の通知
- **✅**: 成功メッセージ、完了時の通知
- **⚠️**: 警告メッセージ、注意が必要な操作

### 3. 禁止事項
- 上記3つ以外の絵文字の使用
- 絵文字のみでのメッセージ表示（必ずテキストと組み合わせる）
- 装飾目的での絵文字使用

## 実装例

### ✅ 良い例
```bash
@echo "✅ デプロイが完了しました"
@echo "⚠️  本番環境の変更内容を確認してください"
@echo "❌ ビルドに失敗しました"
```

### ❌ 悪い例
```bash
@echo "🚀 デプロイを開始します..."  # 装飾目的
@echo "🔧 設定を適用します..."        # 装飾目的
@echo "📊 結果を表示します..."       # 装飾目的
@echo "✅❌⚠️"                      # 複数絵文字
```

## 対象ファイル

この方針は以下のファイルに適用されます：

- `Makefile` - ビルド・デプロイスクリプト
- その他のシェルスクリプト
- ログ出力を含むスクリプト
